Notes for this week

Doing Stuff with Web Things 2.3: Remembering User's Info

  • Computers are stupid, even though they seem smart.
  • Any information stored or shared with a user is called OUTPUT.
  • Anything coming from where it was stored or anything the user enters is called INPUT.
  • The section tag fills the screen all the way across
  • In JS, in order to get the user’s input from any input or text area tag, you use the special word ‘value’.
  • When you want to show something to a user, you use a section and the special word ‘innerHTML’.
  • Input or textarea USE value
  • Section USE innerHTML

Doing Stuff with Web Things 2.4: Making Choices

  • 'else' doesn't do any comparison checks. If you try to do a comparison check with an else, your code will fail.
  • 'else' is only there for when none of the other comparison checks are true.
  • the general rule of thumb, “Don’t do to your users what you don’t like done to you”

Doing Stuff with Web Things 2.5: Groups and Loops

  • A loop is something that makes stuff happen over and over again.
  • The type of loop specifically designed to work with arrays is called a 'for loop.'
  • The standard 'for loop' parts when working with an array are: the 'for' keyword, a variable to keep track of the location you are at in the array (i=0), a check to see if the loop should continue (i less than symbol 7), an update that increases the location tracking variable by one (i++), and the lines of code to execute every time you go through the loop.
  • The '+=' is an operator that takes the variable on the right of it and adds it to the whatever is to the left of it. (ex: 17 += 3 is 20).
