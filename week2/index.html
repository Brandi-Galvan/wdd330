Javascript: Novice to Ninja Chapter 2: Programming Basics

  • Comments:
  • Single line comments start with // and finishes at the end of the line
  • Multi-line comments start with /* and finishes with */
  • JavaScript Grammar:
  • It's considered best practice to combine the two and write each statement on a new line, terminated by a semi-colon, like so:
    const message = 'Hello World!';
    alert(message);
  • A block is a series of statements that are collected together inside curly braces:
    {
    // this is a block containing 2 statements
    const message = 'Hello!';
    alert(message);
    }
  • Reserved Words:
  • The following words are reserved for JavaScript language and cannot be used to name variables:
    abstract, await, boolean, break, byte, case, catch, char, class, const, continue, debugger, default, delete, do, double, else, enum, export, extends, false, final, finally, float, for, function, goto, if, implements, import, in instanceof, int, interface, let, long, native, new, null, package, private, protected, public, return, short, static, super, switch, synchronized, this, throw, throws, transient, true, try, typeof, var, volatile, void, while, with, yield
  • These words are not reserved but really should have been:
    undefined, NaN, Infinity
  • Primitive Date Types:
  • JavaScript has seven different data types. Six of them are primitive data types and are: string, symbol, number, boolean, undefined, and null
  • Any value that isn't listed above is just an object. That includes arrays, functions, and object literals.
  • Variables:
  • It may seem like a restriction to use const, but it actually helps make your programs more predictable if the assignment to variables can't change.
  • variables declared using let and const have block scope, which is discussed in more detail below.
  • Scope:
  • Global Scope is any variable that is declared outside of a block. This means it is accessible everywhere in the program. This is not considered good practice.
  • Local Scope is any variable that is defined inside a block using let or const. They will only be available inside that block and not be accessible outside of that block.
  • Naming Constants + Variables:
  • Constant and variable names can start with any upper or lower-case letter, an underscore, or dollar sign. They can also contain numbers, but cannot start with them.
  • When using multiple words for constant and variable names there are two conventions:
    Camel case starts with a lowercase letter and then each new word is capitalized: firstNameAndLastName
    Underscore separates each new word with an underscore: first_name_and_last_name

Javascript: Novice to Ninja Chapter 3: Arrays, Logic, and Loops

  • Data structures used in JavaScript to store lists of values are called arrays, sets, and maps.
  • An ARRAY is an ordered list of values.
  • The delete operator will remove an item from an array: delete avengers[3];
  • Destructuring an array is the concept of taking values out of an array and presentig them as individual values. ex: const[x,y] = [1,2];
  • To remove the last item from an array we can use the pop() method: avengers.pop()
  • The shift method works in a similar way but it removes the first item in the array: avengers.shift();
  • The push method appends a new value to the end of the array: avengers.push('Thor');
  • The unshift method is similar to the push but it adds a new item to the beginning of the array: avengers.unshift('Captain America');
  • You can MERGE arrays by using the concat() method: avengers.concat(['Hulk','Hawkeye','Black Widow']);
  • The join() methos can be used to turn the array into a string that comprises all the items in the array, separated by commas: avengers.join();
    You can also choose a seperator other than a comma by placing it inside the parentheses: avengers.join('&');
  • Values can be placed into a set using the add method: list.add(1);
  • Multiple items can be placed in a set by repeating the add() method: lits.add(2).add(3).add(4);
