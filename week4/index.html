NTN Chapter 8: Forms

    Forms
  • Forms are made up of a form element that contains form controls such as input fields, select menus and buttons
  • Traditionally, when a form is submitted the information would be sent to a server where the information would be processed using a 'back end' language such as PHP or Ruby.
  • It's becoming more possible to process the information in a form on the 'front end' before it is sent to the server.
    Input Fields
  • Some common types of form control are:
    Input fields: including text, passwords, check boxes, radio buttons, and file uploads.
    Select menus: for drop-down lists of options.
    Textarea elements: for longer text entry.
    Button elements: for submitting and resetting forms.
  • TEXT: The default type of input field is text. The type='text' attribute isn't required but it is advisable to use it as it makes the intended purpose of the field explicit.
    ex: input type='text' id ='donation-amount' name='donationAmount' value='10'
  • PASSWORD: is used to enter passwords or secret information. It looks like type='password'
    ex: input type='password' name='realName' id='realName'
  • CHECKBOX: are creating using input fields type='checkbox'. They are used to select different options that can be checked (true) or left unchecked (false)
    ex: input type='checkbox' id='flight' value='Flight' name='powers'
  • RADIO: these are created using input fields with type='radio'. They allow users to check an option as true but they provide an exclusive choise of options so only one option can be selected.
    ex: input type='radio' name='category' value='Hero' id='hero'
  • HIDDEN: These are not displayed by the browser, but have a 'value' attribute that can contain information that is submitted with the form. type='hidden'
  • FILE INPUT: These are used to upload files and most browsers will provide a brose button or similar that lets users select a file from their file system. type='file'
  • OTHER: There are lots of new input types included, such as 'number', 'tel', and 'color'.
  • NUMBER: these also have optional min and max attributes that can be used to limit the input given.
    ex: input type='number' id='age' name='age' min=0 step=1
