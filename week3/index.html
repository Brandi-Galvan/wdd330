Object Methods Article

  • Functions that are stored in object properties are called "methods".
  • Methods allow objects to "act" like object.doSomething().
  • Methods can reference the object as this.
  • Object methods, "this"
  • OOP (Object-oriented Programming) is when we write our code using obects to represent entities.
  • We can omit the "function" and just write "sayHi()"
  • "this" in methods
  • The value of this is the object "before dot", the one to call the method.
  • "this" is not bound
  • The keyword this behaves unlike most other programming languages. It can be used in any function, even if it's not a method of an object.
  • Calling this without an object would end in an undefined message.

NTN 5: Objects

  • Object Literals
  • An object in JavaScript is a self-contained set of related values and functions. They act as a collection of named properties that map to any JavaScript value such as strings, numbers, booleans, arrays, and functions. If a property's calue is a function, it is known as a method.
  • If a property's value is a function, it is known as a method.
  • An object literal is an object that is created directly in the language by wrapping all its properties and methods in curly braces.
  • Each property in an object literal is a key-value pair, separated by commas.
  • Creating Objects
  • To create an object, start with curly braces {} and assign it to a constant.
  • The shorthand method of creating objects if a property key is the same as a variable name that the property calue is assigned to:
    const ironMan = { name, realName };
  • Accessing Properties
  • ex: superman.name
  • ex2: superman['name']
  • you can use the bracket method when you want to evaluate an expression and use it as the property key:
    superman["real" + " " + "name"]
  • Computer Properties
  • You can create objects with computed property keys, which means the Javascript code can be placed inside square brackets and the property key will be the return value of that code.
  • ex: consthulk = {name: 'Hulk', ['catch + 'Phrase']: 'Hulk Smash!'};
  • now the property key is named 'catchPhrase'.
  • Calling Methods
  • To call a method you can use a dot or bracket notation. It's the same as invoking a function so parentheses need to be placed.
    ex: superman.fly() OR superman['fly']()
  • Checking if properties or methods exist
  • The "in" operator can be used to check whether an object has a particular property.
    'city' in superman; would return false.
  • Adding Properties
  • You can add a new property/method to an object at any time in a program. Just assign a value to the new property.
    ex: superman.city = 'Metropolis';
    Not the superman object has a new city property.
  • Removing Properties
  • You can remove any property from an object with the delete operator.
    ex: delete superman.fly
  • Nested Objects
  • You can nest an object within an object. This is known as a nested object.
  • Here is an example of a nested object:
    const jla = {
    superman: { realName: 'Clark Kent' },
    batman: { realName: 'Bruce Wayne' },}
  • You can access the nested object by referencing each property name in order using either for or bracket notation. ex:
    jla.wonderWoman.realName
    ex 2: jla['flash']['realName']
  • JSON
  • A string representation of the object literal notation above. A couple of differences:
  • Property names must be double-quoted
  • Permitted values are double-quoted strings, numbers, true, false, null, arrays, and objects.
  • Functions are not permitted values
  • PARSE: the parse() method takes a string of data in JSON format and returns a JavaScript object.
    ex: JSON.parse(batman);
  • STRINGIFY: The stringify() method does the opposite, taking a JavaScript object and returning a string of JSON data, as can be seen in the example.
    JSON.stringify(wonderWoman);
  • MATH
  • Rounding methods: Math.ceil(4.2); would return 5.
    Math.floor(4.2); would return 4.
    Math.round(4.5); would return 5.
    Math.pow(3, 2); would return 9.
    Math.pow(27, 1/3); would return 3.
